## 모델 개발 과정

### Phase 1. 초기 시도

**방식** 
15개 객체 동시 학습

**결과**  
이미지 추가
- mAP50: 20% 
- Precision: 30% 
- Recall: 50%

학습저조

**객체 탐지 실패 원인**
1. 클래스별 데이터 부족(50~200개 수준)
2. 라벨링 편차
3. 하이퍼파라미터 최적화 미흡

**문제점 해결 전략**
1.객체 정의 명확화
2. 개별 객체별 단일 클래스 학습


### Phase 2. 단계별 개선 전략1-개별
**방식**
개별 객체 학습

**결과**
✔ 성능 개선 결과  
- mAP50: 20% → 75%
- Precision: 30% → 75%
- Recall: 50% → 80%

→ 총 9개 모델 생성(best03~best13)


### Phase 2. 단계별 개선 전략1-통합
**방식**
개별 객체 학습

**✔ 성능 개선 결과**

- mAP50: 75% → 71% 하락
- Precision: 75% → 60% 하락
- Recall: 80% → 48% 하락


### ⚠ 통합 후 문제 발생

**문제**
클래스 불균형
특징 공간 불일치
예측 충돌

**원인**
편향된 데이터 수 
모델 간 학습 차이
NMS 미흡

**해결방안**
Oversampling, Augmentation
공통 Backbone, Transfer Learning
Soft-NMS, 임계값 조정
